Mastowidget Sample Page

This is your run-of-the-mill sample page to test if Mastowidget is working properly. Just dump this in the same directory as the js and css files (and the mastodon_widget.html file), edit the js accordingly, and pop it open in a browser. Then, you'll see this text and hopefully a widget box elsewhere down the page.

And by "edit the js accordingly", I mean set the accountUrl variable near the top to the account API endpoint on your instance. That will generally be in the form of https://<INSTANCE>/api/v1/accounts/<ID>, where <INSTANCE> is the domain name of your instance and <ID> is your account's numeric user ID.

The easiest way to get that ID (at time of writing) is to log into the web interface, click your username to get your own timeline, and check the URL in your browser. If it says something like https://<INSTANCE>/web/accounts/<ID>, then, well, there you go, that number at the end is your ID. If not, and you can't find it any other way, you may need to ask a moderator or admin of the instance. Mastodon didn't make it easy to find this ID, perhaps with good reason. This also means you might have a hard time displaying anyone else's statuses other than your own.

This is still relatively early code. Most errors won't be displayed, and not everything's going to work yet. Use this at your own risk, I'm not offering any guarantees here.

If need be, tweak the css to taste, and replace the jQuery script line with your favorite flavor of CDN or local file.

Also, just to be clear, this WILL require JavaScript to use. If you or your users have turned it off, there won't be anything displayed. Plan accordingly.
